Technologies

The following are the main technologies that I worked with. Expand the boxes to read about my concrete experience with them. The ordering is based on the amount of experience that I have with the listed technologies, starting with C++ as the programming language that I'm most experienced with. Have fun exploring it!

As my main programming language, I started working with it in my final year of my bachelor studies. Since then, I used it in most of the projects that I did during my journey from an undergrad student up until my time as a PhD student at the Forschungszentrum Informatik (FZI). I'm fascinated how it can be used to do very efficient and hardware related things and at the same time it is very good in abstracting things away. To improve my skills further, I did two online courses. One of them was the C++ for C programmers course at coursera and Udacity's C++ Nanodegree. Currently I mainly use C++17 in my role as a software engineer in the context of telecommunication technology. Doing that, I really like to program multi threaded applications and do proper memory management while I'm working on networking protocol layers like SIP. It is also a common task for me to read RFCs or other standardization manifestos and translate them to C++ code.

Python is the programming language that I favour if C++ is not appropriate to the task that I want to accomplish. I gained my first experiences during my bachelors thesis, where I used it to program a deep learning model to do semantic labeling on images. Since then, I used it to do data analysis and data visualization or to automate things that would be done by hand otherwise. During my time as a PhD student, I programmed a REST service with it using Flask that runs a machine learning model to do object detection on images. In my current role as a software engineer, I use python for most of the things that are not required to be optimized for runtime or memory usage, especially for backend web services. My favourite package manager is anaconda, since it did a lot of machine learning in the past, but if that is not appropriate, I commonly use a virtual environment and install my packages with pip.

Linux is my favourite operating system since the end of my bachelors studies. For me, it is very convenient to have full (sudo) control over the OS and I really like to be in the terminal all day. My experience starts by building projects, goes over system configuration e.g. with systemd and updating configuration files of applications or configureing FTP servers. Logging into remote machines via ssh is also part of my daily business and goes up until creating debian packages for applications to make my code usable for other users that are not developers.

Docker is a very powerful containerization tool to make your applications run on every system that has the docker daemon installed. It is also possible to create development containers, to avoid the "It runs on my machine"-problem. I started using it during my PhD time and currently using it on a daily basis in combination with docker-compose to create software projects with multiple components. It's a common task for me to create Dockerfiles, build containers and run images as a standalone system or within a compose project. I'm also able to configure more complicated things within docker based projects or to talk to the docker API from my code.
